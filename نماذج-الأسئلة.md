# نماذج الأسئلة — المرجع الموحّد لصيغة البيانات

> هذا الملف مرجع **صيغة البيانات** للأنواع **المنفّذة حالياً في الكود** فقط (المذكورة أدناه). أمّا **كتالوج** أنواع الأسئلة (فئاتها وتنظيمها بحسب النطاق) فمرجعه `question-types.md` وهو الحَكَم في وصف الأنواع، و**القواعد الملزمة** (التوزيع، المستوى المعرفي، ربط الهدف، بناء الأنواع الجديدة، نطاق المواد) مرجعها `CLAUDE.md`.

كل أسئلة الدروس تُكتب في `js/questions.js` بالصيغة:

```js
window.QUESTIONS = {
  "معرّف-الدرس": [ /* مصفوفة أسئلة */ ]
};
```

- **المفتاح** = معرّف الدرس (`file`) من `data/index.json` — مثل `"g4s-1-2"`.
- **القيمة** = مصفوفة أسئلة، كل سؤال كائن فيه `type` و`objective` وحقول نوعه.
- الدرس بلا مدخل في `QUESTIONS` يعرض نشاطاً افتراضياً (تسمية أجزاء النبتة).

## الحقول المشتركة لكل سؤال
| الحقل | إلزامي | الوصف |
|------|:---:|------|
| `type` | ✅ | نوع السؤال (أحد الأنواع المنفّذة حالياً أدناه) |
| `objective` | ✅ | نصّ الهدف من وثيقة الأهداف (يُسجَّل مع السؤال) |
| `level` | ✅ | المستوى المعرفي للسؤال — إحدى القيم: `"knowledge"` (معرفة) أو `"application"` (تطبيق) أو `"reasoning"` (استدلال). خاصية للسؤال الفرديّ لا للنوع؛ والتوزيع الإلزامي للأسئلة الأساسية الخمسة في العلوم ٣ معرفة + ١ تطبيق + ١ استدلال (المرجع `CLAUDE.md`) |
| `prompt` | غالباً | نصّ السؤال المعروض (في «صواب وخطأ» يُستخدم `statement` بدلاً منه) |

## القواعد
- **قاعدة التوزيع لكل درس**: مرجعها الملزم `CLAUDE.md` (٥ أساسية إلزامية + إثرائي اختياري = ٥ أو ٦ أسئلة، للعلوم حالياً). لا تُكرَّر هنا حتى لا تتعارض.
- تدرّج من السهل إلى الأصعب.
- كل سؤال مبنيّ على هدف ومستمدّ من الكتاب المدرسي (تأليف أصلي لا نسخ حرفي).

## ① سحب وإفلات — `drag-drop`
الطالب يسحب بطاقات النصّ ويضعها على مواضعها في صورة/رسم.

| الحقل | الوصف |
|------|------|
| `image` | مسار صورة الخلفية (أو `svg` لرسم مضمّن) |
| `fit` | `"height"` للأشكال الطويلة النحيلة (كالهيكل)، أو `"width"` للصور العريضة (كصفّ هياكل جنباً إلى جنب). كلاهما يحافظ على نسبة الصورة وتُحاذى الأهداف عليها. احذفه لتملأ الصورة الملعب |
| `bg` | (اختياري) لون خلفية الملعب |
| `targets[]` | `{ answer, x, y }` — `answer` نصّ البطاقة الصحيحة، و`x`/`y` نسبة مئوية لموضع الهدف على الصورة. البطاقات تُولَّد تلقائياً من `answer` وتُخلط |

```js
{
  type: "drag-drop",
  objective: "4Bh7: يستنتج أنّ العظام ترتبط لتكوّن الهيكل العظمي",
  level: "application",
  prompt: "اسحب اسم كل عظم إلى مكانه في الهيكل.",
  image: "images/skeleton.png",
  fit: "height",
  bg: "#fdf9ee",
  targets: [
    { answer: "الجُمجمة",    x: 50, y: 7 },
    { answer: "ألواح الكتف", x: 36, y: 17 },
    { answer: "الأضلاع",     x: 53, y: 27 },
    { answer: "عظام الورك",  x: 50, y: 46 },
    { answer: "عظمة الفخذ",  x: 42, y: 61 },
    { answer: "قصبة الساق",  x: 58, y: 83 }
  ]
}
```

## ② توصيل — `matching`
الطالب يضغط عنصراً من العمود الأيمن ثم ما يناسبه من الأيسر.

| الحقل | الوصف |
|------|------|
| `pairs[]` | `{ a, b }` — `a` العنصر، `b` ما يوصَل به. تُخلط الأعمدة تلقائياً |

```js
{
  type: "matching",
  objective: "4Bh6: يلاحظ أنّ للعظام أشكالاً وأحجاماً مختلفة",
  level: "knowledge",
  prompt: "صِل كل عظم بنوعه.",
  pairs: [
    { a: "عظمة الفخذ",   b: "عظم طويل" },
    { a: "عظام الأصابع", b: "عظم قصير" },
    { a: "الجُمجمة",     b: "عظم مسطّح" },
    { a: "الفقرة",       b: "عظم غير منتظم" }
  ]
}
```

## ③ اختيار من متعدد — `mcq`
سؤال بـ٣–٤ خيارات، واحد صحيح. الخيارات تُخلط تلقائياً.

| الحقل | الوصف |
|------|------|
| `options[]` | مصفوفة الخيارات النصّية |
| `answer` | فهرس الخيار الصحيح في `options` (يبدأ من 0) |

```js
{
  type: "mcq",
  objective: "4Bh6: يلاحظ أنّ للعظام أشكالاً وأحجاماً مختلفة",
  level: "knowledge",
  prompt: "كم عدد العظام في هيكل جسم الإنسان؟",
  options: ["١٠٦ عظمة", "٢٠٦ عظمة", "٣٠٦ عظمة", "٤٠٦ عظمة"],
  answer: 1
}
```

## ④ صواب وخطأ — `true-false`
عبارة يحكم عليها الطالب بصواب أو خطأ.

| الحقل | الوصف |
|------|------|
| `statement` | نصّ العبارة (يُعرض مكان `prompt`) |
| `answer` | `true` إن كانت العبارة صحيحة، `false` إن كانت خاطئة |

```js
{
  type: "true-false",
  objective: "4Bh7: يستنتج أنّ العظام ترتبط لتكوّن الهيكل العظمي",
  level: "knowledge",
  statement: "الجُمجمة تتكوّن من عظمة واحدة فقط.",
  answer: false
}
```

## ⑤ تحديد الأجزاء — `hotspot`
الطالب ينقر على الموضع الصحيح في صورة/رسم.

| الحقل | الوصف |
|------|------|
| `image` | مسار الصورة (أو `svg`) — تُوضع دائماً داخل غلاف يحافظ على نسبتها |
| `bg` | (اختياري) لون خلفية الملعب |

```js
  bg: "#fdf9ee",
  spot: { x: 53, y: 27, r: 11 }
}
```

## ⑥ الترتيب التسلسلي — `sequence`
الطالب يرتّب بطاقات مبعثرة بالتسلسل الصحيح **بسحبها لتغيير أماكنها** (فأرة + لمس على السبورة). البطاقات تُخلط تلقائياً عند العرض، وتنزاح لإفساح مكان البطاقة المسحوبة.

**الآلية المعتمدة:** الأرقام (١، ٢، ٣…) تظهر في **عمود ثابت خارج الصناديق بجانبها** ولا تتحرك؛ الطالب يسحب كل بطاقة إلى جوار رقمها الصحيح، وعند اكتمال الترتيب الصحيح تتطابق البطاقات مع أرقامها. التحقّق يقارن ترتيب البطاقات الحالي بترتيب `steps`.

| الحقل | الوصف |
|------|------|
| `steps[]` | مصفوفة نصوص الخطوات **بالترتيب الصحيح**. تُعرض الأرقام في عمود ثابت خارج البطاقات، والتحقّق يقارن ترتيب البطاقات الحالي بهذا الترتيب |

```js
{
  type: "sequence",
  objective: "4Bh10: يتتبّع مسار الدواء في الجسم حتى وصوله إلى موضع الألم",
  level: "reasoning",
  prompt: "رتّب رحلة الدواء في الجسم من البداية إلى النهاية.",
  steps: [
    "تناول الدواء",
    "وصوله إلى المعدة",
    "انتقاله إلى الدم",
    "وصوله إلى مكان الألم"
  ]
}
```

## ⑦ التصنيف في مجموعات — `classify`
الطالب يسحب عناصر متعددة من بنك العناصر إلى **صندوق مجموعتها الصحيحة**. الصناديق جنباً إلى جنب بلا صورة مركزية ولا خطوط واصلة (النمط المعتمد للتصنيف). العناصر تُخلط تلقائياً، ويمكن سحبها بين المجموعات أو إعادتها للبنك. يعمل بالسحب (فأرة + لمس على السبورة).

**المواصفات المعتمدة:** كل صندوق = **عنوان المجموعة** (شريط علوي شفّاف) فوق **منطقة إلقاء بيضاء** تُوضع فيها العناصر. عند التحقّق: العنصر في مجموعته الصحيحة يتحوّل **أخضر**، والخاطئ **أحمر**. عناصر كبيرة تناسب اللمس على السبورة، واتجاه RTL.

| الحقل | الوصف |
|------|------|
| `groups[]` | `{ name, items }` — `name` اسم المجموعة (عنوان الصندوق)، و`items` مصفوفة العناصر التي تنتمي إليها. عند التحقّق يُقارن موضع كل عنصر باسم مجموعته الصحيحة |

```js
{
  type: "classify",
  objective: "4Bh1: يستنتج أنّ للإنسان وبعض الحيوانات هيكلاً عظميًّا داخل الجسم",
  level: "reasoning",
  prompt: "صنّف الحيوانات حسب هيكلها.",
  groups: [
    { name: "فقاريات",   items: ["إنسان", "سمكة", "طائر"] },
    { name: "لافقاريات", items: ["حلزون", "دودة", "حشرة"] }
  ]
}
```

## ⑧ ملء الفراغ بالسحب — `fill-blank`
جملة (أو أكثر) فيها فراغات، والطالب يسحب الكلمة المناسبة من بنك الكلمات إلى كل فراغ. يُوضع في البنك كلمات مموّهة (distractors) لزيادة التحدّي. يعمل بالسحب (فأرة + لمس)؛ **نقر الفراغ الممتلئ يفرّغه** ويعيد كلمته للبنك. عند التحقّق يتحوّل الفراغ الصحيح **أخضر** والخاطئ **أحمر**.

| الحقل | الوصف |
|------|------|
| `text` | نصّ الجملة، وكل فراغ يُكتب كعلامة `{}` بالترتيب. مثال: `"النبات يحتاج {} و {} لينمو"` |
| `answers[]` | الكلمات الصحيحة **بترتيب الفراغات** (`answers[0]` للفراغ الأول وهكذا) |
| `distractors[]` | (اختياري) كلمات مموّهة تُضاف للبنك ولا تنتمي لأي فراغ |

```js
{
  type: "fill-blank",
  objective: "4Bh11: يدرك أهمية أخذ الدواء بالجرعة والوقت المناسبين وبإشراف بالغ",
  level: "application",
  prompt: "أكمل الجمل بسحب الكلمات المناسبة.",
  text: "يجب أخذ الدواء بـ {} المحددة، وفي {} المناسب، ولا نتناول دواءً دون إشراف {}",
  answers: ["الجرعة", "الوقت", "شخص بالغ"],
  distractors: ["اللون", "الحجم"]
}
```

## ⑨ الاستبعاد (الدخيل) — `exclude`
عناصر تشترك في صفة واحدة وبينها عنصر **دخيل** لا ينتمي، والطالب يضغط الدخيل. الخيارات تُخلط تلقائياً. النقر على عنصر منتمٍ يعلّمه **أحمر** ويعطّله (تصحيح ذاتي)، والنقر على الدخيل يعلّمه **أخضر** ويُنهي السؤال. يُنمّي المقارنة والتفكير النقدي.

| الحقل | الوصف |
|------|------|
| `options[]` | مصفوفة العناصر النصّية (منها الدخيل والبقية تشترك في صفة) |
| `answer` | فهرس العنصر الدخيل في `options` (يبدأ من 0) |
| `reason` | (اختياري) سبب عدم انتماء الدخيل، يُعرض في رسالة النجاح لترسيخ المفهوم |

```js
{
  type: "exclude",
  objective: "يستخدم صفات الكائنات ليميّز الدخيل عن مجموعة تشترك في صفة",
  level: "reasoning",
  prompt: "أيّها لا ينتمي؟",
  options: ["عصفور", "نسر", "بطة", "قطة"],
  answer: 3,
  reason: "القطة ليست طائراً"
}
```

## ⑩ ترتيب الحروف — `arrange`
حروف مبعثرة والطالب يسحب كل حرف إلى خانته بالترتيب (قراءة **يمين→يسار**) لتكوين الكلمة العلمية. الحروف تُعرض **منفصلة (مفردة)** أثناء الترتيب (أوضح للأطفال)، وعند اكتمال الترتيب الصحيح تُكشف **الكلمة كاملة متّصلة** ليرى الطفل شكلها النهائي. تُحرّك بطاقة الحرف نفسها إلى الخانة (يدعم الحروف المكرّرة)، ونقر الخانة الممتلئة يعيد حرفها للبنك. يعمل بالسحب (فأرة + لمس). عند التحقّق: الحرف في مكانه الصحيح يتحوّل **أخضر** والخاطئ **أحمر**. يرسّخ المصطلحات العلمية وإملاءها.

| الحقل | الوصف |
|------|------|
| `word` | الكلمة الصحيحة. تُشتقّ منها الخانات والترتيب الصحيح (قراءة يمين→يسار). رسالة النجاح تعرضها |
| `letters` | (اختياري) الحروف المبعثرة في البنك. احذفه لتُشتقّ تلقائياً من حروف `word`. استخدمه لإضافة حرف مموّه أو للتحكّم في تقطيع كلمة فيها حروف متّصلة |

```js
{
  type: "arrange",
  objective: "يرسّخ كتابة مصطلح «المادة» بترتيب حروفه ترتيباً صحيحاً",
  level: "knowledge",
  prompt: "رتّب الحروف لتكوين الكلمة الصحيحة.",
  word: "مادة"
}
```

## ⑪ الخريطة الذهنية الناقصة — `mindmap`
عقدة مركزية (`center`) تتفرّع إلى فروع، لكلّ فرع **عنوان ثابت** (`label`) و**خانة فارغة** يسحب إليها الطالب الكلمة الصحيحة من بنك الكلمات لبناء خريطة المفاهيم. تُرسم خطوط منحنية من المركز إلى كل فرع وتُحدَّث تلقائياً عند تغيّر الحجم. تُحرّك بطاقة الكلمة نفسها إلى الخانة (تدعم التكرار)، ونقر الخانة الممتلئة يعيد كلمتها للبنك. يعمل بالسحب (فأرة + لمس). عند التحقّق: الخانة الصحيحة **خضراء** والخاطئة **حمراء**. يبني التفكير المنظومي وربط المفاهيم.

| الحقل | الوصف |
|------|------|
| `center` | نصّ العقدة المركزية (المفهوم الرئيس) |
| `branches[]` | `{ label, answer }` — `label` عنوان الفرع الثابت، و`answer` الكلمة الصحيحة التي تُسحب إلى خانته |
| `distractors[]` | (اختياري) كلمات مموّهة تُضاف للبنك ولا تنتمي لأيّ فرع |

```js
{
  type: "mindmap",
  objective: "يربط كل حالة من حالات المادة بمثال مناسب لها",
  level: "application",
  prompt: "أكمل الخريطة الذهنية بسحب المثال المناسب إلى كل فرع.",
  center: "حالات المادة",
  branches: [
    { label: "صلبة",  answer: "طوب" },
    { label: "سائلة", answer: "ماء" },
    { label: "غازية", answer: "هواء" }
  ],
  distractors: ["نار", "ضوء"]
}
```

## ⑫ اكتشف الخطأ — `find-error`
صورة فيها **خطأ علمي واحد** (كائن في موطن خاطئ، أو عنصر في غير موضعه)، والطالب يضغط على موضع الخطأ. نفس آليّة **النقطة الساخنة** بصرياً: نقرة داخل دائرة الخطأ (`spot`) تفوز، وخارجها تُشجّع على التدقيق. الصورة توضَع دائماً داخل غلاف يحافظ على نسبتها. ينمّي الملاحظة والتفكير النقدي.

| الحقل | الوصف |
|------|------|
| `image` | مسار الصورة (أو `svg`) |
| `fit` | (اختياري) `"width"` للصور العريضة؛ يُحاذى الغلاف على نسبة الصورة |
| `bg` | (اختياري) لون خلفية الملعب |
| `spot` | `{ x, y, r }` — مركز موضع الخطأ (`x`/`y` نسبة مئوية) ونصف القطر المقبول `r` (نسبة مئوية، الافتراضي 10) |

```js
{
  type: "find-error",
  objective: "يميّز الكائن الموجود في موطن غير مناسب له",
  level: "reasoning",
  prompt: "في هذا المشهد كائن وُضع في الموطن الخطأ — اضغط على الخطأ.",
  image: "images/خطأ-الموطن.png",
  spot: { x: 30, y: 26, r: 13 }
}
```

## ⑬ السؤال الصوتي — `audio-q`
يُشغَّل صوت (صوت حيوان مثلاً) بزر **«استمع»**، والطالب يختار مصدره من **صور الخيارات** (تُخلط تلقائياً كنمط الاختيار من متعدد). النقر على الصورة الصحيحة → فوز، والخاطئة → تُعطّل ويُشجَّع على الاستماع ثانيةً. صوت السؤال يُشغَّل بضغطة صريحة على الزر و**لا يخضع لكتم أصوات التغذية الراجعة** (correct/wrong) كي تبقى مسموعة. إن تعذّر تحميل صورة خيار تُخفى وتبقى **تسميتها** ظاهرة (تدرّج سليم قبل توليد الصور). يدرّب حاسة السمع.

| الحقل | الوصف |
|------|------|
| `sound` | مسار ملف الصوت في `audio/` (mp3 أو wav) |
| `options[]` | `{ image, label }` — `image` مسار صورة الخيار في `images/`، و`label` اسمه (يظهر كتسمية وكبديل إن غابت الصورة) |
| `answer` | فهرس الخيار الصحيح في `options` (يبدأ من 0) |

## ⑭ التكبير التدريجي — `zoom-reveal`
صورة **واحدة** عالية الوضوح تبدأ **مقرّبة جداً** ثم تتّسع تدريجياً إلى حجمها الكامل بعد الضغط على «ابدأ التكبير»، والطالب يخمّن **«ما هذا؟»** من الخيارات (تُخلط كنمط الاختيار من متعدد). **التخمين المبكر** (والصورة أكثر تقريباً) يمنح **نجوماً أكثر** (من ٥ عند أقصى تقريب إلى ١ عند الحجم الكامل)؛ عدّاد النقاط يظهر الرصيد الحالي مباشرةً. الخيار الصحيح → فوز بعدد النجوم الحالي، والخاطئ → يُعطّل ويُشجَّع على الانتظار. لا حاجة لصور متعددة — الكود يتحكّم بالتقريب. مثالي لبدء الدرس أو ختامه.

| الحقل | الوصف |
|------|------|
| `image` | مسار الصورة الواحدة عالية الوضوح (يُفضَّل أن يملأ الكائن الإطار ويتوسّطه) |
| `options[]` | مصفوفة الخيارات النصّية (تُخلط) |
| `answer` | فهرس الخيار الصحيح في `options` (يبدأ من 0) |
| `maxZoom` | (اختياري) مقدار التقريب الابتدائي (الافتراضي 6) |
| `seconds` | (اختياري) زمن الاتّساع الكامل بالثواني (الافتراضي 9) |

```js
{
  type: "zoom-reveal",
  objective: "يتعرّف على الحشرة من تفاصيلها الدقيقة",
  level: "knowledge",
  prompt: "ما هذا الكائن؟ خمّن مبكراً كي تربح نجوماً أكثر.",
  image: "images/حشرة-تكبير.png",
  options: ["حشرة", "زهرة", "حجر", "ورقة شجر"],
```

## ⑮ التلوين بالتعليمات — `color`
رسم **SVG بأجزاء منفصلة** (كل جزء عنصر `<g class="cpart" data-name="…">` قابل للتلوين — بنيته على نمط مجموعات `<g>` في دورة حياة النبات، **لا صورة نقطية خارجية ولا برومبت**). الطالب يختار لوناً من **لوحة ألوان كبيرة** ثم يضغط الجزء فيتلوّن بالكامل. عند التحقّق: كل جزء مطلوب باللون الصحيح → **فوز**، وإلّا يُحاط الجزء الخاطئ بإطار أحمر متقطّع ويُعرض عدد الصحيح. **الأجزاء غير المذكورة في `parts` تبقى حرّة** (تلوينها أو تركها لا يؤثّر في النتيجة). عناصر كبيرة تناسب اللمس على السبورة واتجاه RTL، والصوت (correct.mp3/wrong.mp3) عبر `qWin/qFail` خاضعاً لزرّ الكتم العامّ.

**بنية الرسم:** كل جزء قابل للتلوين هو `<g class="cpart" data-name="اسم الجزء" id="part-…">` يحوي أشكاله؛ تُترك الأشكال **بلا `fill` خاصّ بها** لترث لون المجموعة (الكود يضبط `fill` على `<g>` عند الضغط). التفاصيل الثابتة (كعيون الجمجمة، خطوط الأضلاع) تُعطى `fill` أو `fill="none"` صريحاً فلا يطالها التلوين.

| الحقل | الوصف |
|------|------|
| `svg` | رسم SVG مضمّن، كل جزء قابل للتلوين `<g class="cpart" data-name="…">`. `data-name` يُطابق اسم الجزء في `parts` |
| `palette[]` | `{ name, color }` — ألوان اللوحة المتاحة (`name` اسم اللون، `color` قيمته السداسية) |
| `parts[]` | `{ name, color }` — لكل جزء **مطلوب** اسمه (يطابق `data-name`) ولونه الصحيح. الأجزاء غير المذكورة هنا حرّة |
| `bg` | (اختياري) لون خلفية الملعب |

```js
  bg: "#fdf9ee",
  palette: [
    { name: "أزرق", color: "#2f6fb0" },
    { name: "أحمر", color: "#cf3b3b" },
    { name: "أخضر", color: "#3e9b4f" }
  ],
  parts: [
    { name: "الجمجمة",       color: "#2f6fb0" },
    { name: "القفص الصدري",  color: "#cf3b3b" },
    { name: "العمود الفقري", color: "#3e9b4f" }
  ],
  svg: `<svg viewBox="0 0 220 430" xmlns="http://www.w3.org/2000/svg">
    <g class="cpart" data-name="الجمجمة" id="part-skull"> … </g>
    <g class="cpart" data-name="القفص الصدري" id="part-ribs"> … </g>
    <g class="cpart" data-name="العمود الفقري" id="part-spine"> … </g>
```

## ⑯ البازل (تركيب الصورة) — `puzzle`
**صورة واحدة كاملة** يقسّمها الكود برمجياً إلى شبكة `grid.cols × grid.rows` قطعاً (٣×٣ مثلاً) عبر خاصية `background-position` — **لا حاجة لصور متعددة، صورة واحدة تكفي**. تُبعثَر القطع في صينية سفلية، ويعيد الطالب ترتيبها بالسحب (فأرة + لمس) إلى لوح الشبكة؛ إفلات قطعة على خانة مشغولة يبدّل ساكنها إلى الصينية. عند التحقّق: كل قطعة في خانتها الصحيحة → **مكافأة** (تختفي الفواصل والحدود فتظهر الصورة صورةً واحدة متّصلة) وصوت `correct.mp3`، وإلّا يُحاط كل خانة خاطئة بإطار أحمر ويُعرض عدد الصحيح. لوح الشبكة `direction:ltr` كي تطابق أعمدةُ الشبكة أعمدةَ الصورة (الصورة محتوى بصري لا نصّ) بينما تبقى بقيّة الواجهة RTL، والقطع كبيرة تناسب اللمس على السبورة، والصوت (`correct.mp3`/`wrong.mp3`) عبر `qWin/qFail` خاضعاً لزرّ الكتم العامّ.

| الحقل | الوصف |
|------|------|
| `image` | مسار **صورة واحدة كاملة** (يُفضَّل أن يملأ الكائن الإطار ويتوسّطه)؛ الكود يقسّمها ويبعثرها. اللوح يأخذ نسبة الصورة الحقيقية تلقائياً |
| `grid` | `{ cols, rows }` — عدد أعمدة وصفوف التقسيم (الافتراضي `{ cols: 3, rows: 3 }`). عدد القطع = `cols × rows` |
| `bg` | (اختياري) لون خلفية اللوح |

```js
{
  type: "puzzle",
  objective: "يتعرّف على الشكل العام للهيكل العظمي وأجزائه",
  level: "application",
  prompt: "ركّب صورة الهيكل العظمي: اسحب كل قطعة إلى مكانها الصحيح.",
  image: "images/هيكل-بازل.png",
  bg: "#fdf9ee",
  grid: { cols: 3, rows: 3 }
}
```

> ملاحظة: يحتاج هذا النوع **صورة واحدة** فقط (برومبتها في `برومبتات-الرسومات.docx` بأسلوب باقي البرومبتات ومرساة الأسلوب المسطّح)؛ الكود يتكفّل بالتقسيم والبعثرة وإعادة التركيب. انظر المثال الحيّ في `js/questions.js` تحت `"g4s-1-2"`.

## ⑰ الشريط المتدرّج — `slider`
**شريط أفقيّ** بمؤشّر دائريّ كبير يسحبه الطالب لتحديد **قيمة رقمية** على **تدريج مرئيّ واضح بأرقام كبيرة** (كمسطرة/مقياس حرارة). يبدأ المؤشّر في **منتصف الشريط** (لا عند الإجابة)، ويعمل بالسحب (فأرة + لمس على السبورة)، كما **ينقر الطالب أيّ موضع على الشريط فينتقل المؤشّر إليه مباشرةً**. تُعرض القيمة المختارة بأرقام كبيرة أعلى الشريط لحظةً بلحظة. عند التحقّق: إذا كان **الفرق بين القيمة المختارة والصحيحة ≤ `tolerance`** يفوز الطالب (يتحوّل المؤشّر أخضر)، وإلّا يُعرض تلميح إن كانت قيمته **أصغر** أو **أكبر** من المطلوب. يعلّم التقدير والقياس للمفاهيم الرقمية.

**هامش القبول (`tolerance`)**: لأنّ اللمس على السبورة الذكية **غير دقيق**، يُسمح بهامش (±) حول القيمة الصحيحة كي لا يُحرَم الطالب لخطأ لمسٍ بسيط. مثال: `answer: 100, tolerance: 2` يقبل كلّ قيمة من ٩٨ إلى ١٠٢.

**الاتجاه**: الشريط `dir:ltr` كي **تتصاعد الأرقام يساراً→يميناً** كمسطرة (الأدنى يساراً والأقصى يميناً)، بينما تبقى بقيّة الواجهة (النصّ والأزرار) RTL. عناصر كبيرة (مؤشّر ٤٦px وأرقام واضحة) تناسب اللمس على السبورة، والصوت (`correct.mp3`/`wrong.mp3`) عبر `qWin/qFail` خاضعاً لزرّ الكتم العامّ.

| الحقل | الوصف |
|------|------|
| `min` | أدنى قيمة على الشريط (طرفه الأيسر) |
| `max` | أقصى قيمة على الشريط (طرفه الأيمن) |
| `answer` | القيمة الصحيحة المطلوبة |
| `tolerance` | (اختياري) هامش القبول ± حول `answer` (الافتراضي 0). يُنصح به على السبورة لعدم دقّة اللمس |
| `unit` | (اختياري) لاحقة تُعرض بعد كل رقم (مثل `"°"` للحرارة أو `"cm"`) |
| `step` | (اختياري) خطوة تقريب القيمة عند السحب (الافتراضي 1) |
| `ticks` | (اختياري) المسافة بين علامات التدريج الكبيرة وأرقامها. احذفه ليُشتقّ تلقائياً نحو ٦ فترات |

```js
{
  type: "slider",
  objective: "يحدد درجة غليان الماء على مقياس الحرارة",
  level: "knowledge",
  prompt: "اسحب المؤشر إلى درجة غليان الماء على مقياس الحرارة.",
  min: 0,
  max: 120,
  answer: 100,
  tolerance: 2,
  unit: "°",
  ticks: 20
}
```

> ملاحظة: هذا النوع لا يحتاج صورة ولا برومبت رسم؛ الشريط والتدريج مبنيّان بالكود بالكامل من `min`/`max`/`ticks`. انظر المثال الحيّ في `js/questions.js` تحت `"g4s-3-6"`.

## ⑱ بطاقات الذاكرة — `memory`
شبكة **بطاقات مقلوبة**؛ كل زوج `{ a, b }` يولّد **بطاقتين تشتركان في مفتاح** (فهرس الزوج) وتُخلط كل البطاقات مقلوبة. يقلب الطالب **بطاقتين في كل دور**: إن تطابقتا (نفس المفتاح) **تبقيان مكشوفتين** بلون النجاح (صوت `correct.mp3` + نجمة)، وإن لم تتطابقا **تُقلبان ثانيةً بعد لحظة** (صوت `wrong.mp3`). ينتهي السؤال عند **كشف كل الأزواج**. البطاقات كبيرة مربّعة تناسب اللمس على السبورة، والواجهة RTL، والصوت (`correct.mp3`/`wrong.mp3`) عبر `playCorrectSound`/`qWin` خاضعاً لزرّ الكتم العامّ.

**النسخة الحالية نصّية** (مطابقة **كلمة بكلمة** مرتبطة، مثل طائر ← صفته) — **لا تحتاج صوراً ولا برومبت رسم**. عدد الأعمدة يُشتقّ تلقائياً من عدد البطاقات (٦ بطاقات ← ٣ أعمدة، ٨ ← ٤).

| الحقل | الوصف |
|------|------|
| `pairs[]` | `{ a, b }` — طرفا الزوج المتطابق (`a` و`b`). كل زوج يولّد بطاقتين تُكشفان معاً عند تطابق مفتاحهما. عدد البطاقات = `pairs.length × 2` (٤ أزواج = ٨ بطاقات) |

```js
{
  type: "memory",
  objective: "يربط كل طائر بصفته المميزة",
  level: "knowledge",
  prompt: "اعثر على الأزواج المتطابقة: اقلب بطاقتين في كل دور لتطابق كل طائر بصفته.",
  pairs: [
    { a: "النعامة", b: "أكبر طائر" },
    { a: "البطريق", b: "لا يطير" },
    { a: "الطنّان",  b: "أصغر طائر" },
    { a: "البومة",  b: "تصطاد ليلاً" }
  ]
}
```

> ملاحظة: النسخة الحالية نصّية بالكامل (بطاقتا كل زوج نصّان) فلا تحتاج صوراً؛ يمكن لاحقاً توسعة النوع لمطابقة **صورة ↔ اسم** بإضافة صور للبطاقات. انظر المثال الحيّ في `js/questions.js` تحت `"g4s-2-1"` (الطيور المُدهشة).

## ⑲ العدسة المكبّرة — `lens`
**نوع خاصّ بالعلوم** (انظر قسمه في `question-types.md`) — حالته: **مبني، قيد الاختبار**.
صورتان PNG **متطابقتا الأبعاد**: العلوية (`image`) هي **المشهد الظاهر**، والسفلية (`hidden`) هي **الطبقة الخفية** تحته. العدسة دائرة `clip-path` قطرها **~160px** بمقبض واضح، يسحبها الطالب (فأرة + لمس معاً — الاستخدام الأساسي سبورة تفاعلية بالأصابع) فتكشف الطبقة السفلية أينما تحرّكت. العناصر المخفية **نقاط ساخنة** (`spots`): يضغط الطالب العنصر **وهو ظاهر داخل العدسة** — الصحيح = صوت النجاح المعتمد + **يثبت العنصر مكشوفاً مع تسميته** + تقدّم عدّاد «اكتشفت ٢ من ٥»؛ والخاطئ = تغذية الخطأ المعتمدة (`qFail`: اختناق محرّك الصاروخ، و`wrong.mp3` في واجهة بلا صاروخ). يكتمل السؤال **باكتشاف كل العناصر**. واجهة عربية RTL وعناصر كبيرة للسبورة.

**اصطلاح تسمية الصورتين:** الصورة العلوية باسم موضوع السؤال (بشَرطات بلا مسافات)، والسفلية **بنفس الاسم + لاحقة `-خفي`** — مثال: `طفل-أشعة.png` و`طفل-أشعة-خفي.png`.

| الحقل | الوصف |
|------|------|
| `image` | مسار الصورة **العلوية** (المشهد الظاهر) |
| `hidden` | مسار الصورة **السفلية** (الطبقة الخفية) — بنفس أبعاد العلوية تماماً كي تتطابق الطبقتان |
| `fit` | (اختياري) `"width"` للصور العريضة (كنمط hotspot) |
| `bg` | (اختياري) لون خلفية الملعب |
| `spots[]` | `{ label, x, y, r, alt? }` — العناصر المخفية: `label` التسمية التي تظهر عند الاكتشاف، و`x`/`y` مركز العنصر (نسبة مئوية على الصورة)، و`r` نصف القطر المقبول (نسبة مئوية، الافتراضي 10). `alt` (اختياري) = `{ x, y, r? }` دائرة **بديلة** لنفس العنصر تُقبل أيضاً — للأعضاء المزدوجة المتماثلة (كالذراعين والساقين) كي لا تُحتسب الجهة الأخرى خطأً؛ التسمية تظهر عند الدائرة الرئيسة. `labelBelow: true` (اختياري) يعرض التسمية **أسفل** العنصر بدل أعلاه — للعناصر المتجاورة كي لا تتراكب تسمياتها |

```js
{
  type: "lens",
  objective: "4Bh1: يستنتج أنّ للإنسان وبعض الحيوانات هيكلاً عظميًّا داخل الجسم",
  level: "application",
  prompt: "اسحب عدسة الأشعة السينية فوق جسم الطفل، واضغط على كل جزء من هيكله العظمي يظهر داخلها.",
  image: "images/طفل-أشعة.png",
  hidden: "images/طفل-أشعة-خفي.png",
  bg: "#fdf9ee",
  spots: [
    { label: "الجمجمة",       x: 49.5, y: 23.5, r: 8 },
    { label: "القفص الصدري",  x: 50,   y: 43,   r: 6 },
    { label: "العمود الفقري", x: 50,   y: 53,   r: 5.5 },
    { label: "عظام الذراعين", x: 40.5, y: 55,   r: 8,   alt: { x: 59.5, y: 55 },   labelBelow: true },
    { label: "عظام الساقين",  x: 45.5, y: 72.5, r: 9,   alt: { x: 53,   y: 72.5 }, labelBelow: true }
  ]
}
```

> ملاحظة: يحتاج هذا النوع **صورتين متطابقتي الأبعاد** (برومبتاهما بأسلوب ملف البرومبتات ومرساة الأسلوب المسطّح، مع تطابق الوضعية والإطار بين الصورتين). انظر المثال الحيّ في `js/questions.js` تحت `"g4s-1-1"` (الهياكل العظمية).

## كيف تُحدّد نِسَب `x`/`y` على صورة؟
