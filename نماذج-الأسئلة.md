# نماذج الأسئلة — المرجع الموحّد لصيغة البيانات

كل أسئلة الدروس تُكتب في `js/questions.js` بالصيغة:

```js
window.QUESTIONS = {
  "معرّف-الدرس": [ /* مصفوفة أسئلة */ ]
};
```

- **المفتاح** = معرّف الدرس (`file`) من `data/index.json` — مثل `"g4s-1-2"`.
- **القيمة** = مصفوفة أسئلة، كل سؤال كائن فيه `type` و`objective` وحقول نوعه.
- الدرس بلا مدخل في `QUESTIONS` يعرض نشاطاً افتراضياً (تسمية أجزاء النبتة).

## الحقول المشتركة لكل سؤال
| الحقل | إلزامي | الوصف |
|------|:---:|------|
| `type` | ✅ | نوع السؤال (أحد الخمسة أدناه) |
| `objective` | ✅ | نصّ الهدف من وثيقة الأهداف (يُسجَّل مع السؤال) |
| `prompt` | غالباً | نصّ السؤال المعروض (في «صواب وخطأ» يُستخدم `statement` بدلاً منه) |

## القواعد (من CLAUDE.md)
- ٤ إلى ٦ أسئلة لكل درس، على نوعين مختلفين على الأقل.
- تدرّج من السهل إلى الأصعب.
- كل سؤال مبنيّ على هدف ومستمدّ من الكتاب المدرسي (تأليف أصلي لا نسخ حرفي).

## ① سحب وإفلات — `drag-drop`
الطالب يسحب بطاقات النصّ ويضعها على مواضعها في صورة/رسم.

| الحقل | الوصف |
|------|------|
| `image` | مسار صورة الخلفية (أو `svg` لرسم مضمّن) |
| `fit` | `"height"` للأشكال الطويلة النحيلة (كالهيكل)، أو `"width"` للصور العريضة (كصفّ هياكل جنباً إلى جنب). كلاهما يحافظ على نسبة الصورة وتُحاذى الأهداف عليها. احذفه لتملأ الصورة الملعب |
| `bg` | (اختياري) لون خلفية الملعب |
| `targets[]` | `{ answer, x, y }` — `answer` نصّ البطاقة الصحيحة، و`x`/`y` نسبة مئوية لموضع الهدف على الصورة. البطاقات تُولَّد تلقائياً من `answer` وتُخلط |

```js
{
  type: "drag-drop",
  objective: "4Bh7: يستنتج أنّ العظام ترتبط لتكوّن الهيكل العظمي",
  prompt: "اسحب اسم كل عظم إلى مكانه في الهيكل.",
  image: "images/skeleton.png",
  fit: "height",
  bg: "#fdf9ee",
  targets: [
    { answer: "الجُمجمة",    x: 50, y: 7 },
    { answer: "ألواح الكتف", x: 36, y: 17 },
    { answer: "الأضلاع",     x: 53, y: 27 },
    { answer: "عظام الورك",  x: 50, y: 46 },
    { answer: "عظمة الفخذ",  x: 42, y: 61 },
    { answer: "قصبة الساق",  x: 58, y: 83 }
  ]
}
```

## ② توصيل — `matching`
الطالب يضغط عنصراً من العمود الأيمن ثم ما يناسبه من الأيسر.

| الحقل | الوصف |
|------|------|
| `pairs[]` | `{ a, b }` — `a` العنصر، `b` ما يوصَل به. تُخلط الأعمدة تلقائياً |

```js
{
  type: "matching",
  objective: "4Bh6: يلاحظ أنّ للعظام أشكالاً وأحجاماً مختلفة",
  prompt: "صِل كل عظم بنوعه.",
  pairs: [
    { a: "عظمة الفخذ",   b: "عظم طويل" },
    { a: "عظام الأصابع", b: "عظم قصير" },
    { a: "الجُمجمة",     b: "عظم مسطّح" },
    { a: "الفقرة",       b: "عظم غير منتظم" }
  ]
}
```

## ③ اختيار من متعدد — `mcq`
سؤال بأربعة خيارات، واحد صحيح. الخيارات تُخلط تلقائياً.

| الحقل | الوصف |
|------|------|
| `options[]` | مصفوفة الخيارات النصّية |
| `answer` | فهرس الخيار الصحيح في `options` (يبدأ من 0) |

```js
{
  type: "mcq",
  objective: "4Bh6: يلاحظ أنّ للعظام أشكالاً وأحجاماً مختلفة",
  prompt: "كم عدد العظام في هيكل جسم الإنسان؟",
  options: ["١٠٦ عظمة", "٢٠٦ عظمة", "٣٠٦ عظمة", "٤٠٦ عظمة"],
  answer: 1
}
```

---

## ④ صواب وخطأ — `true-false`
عبارة يحكم عليها الطالب بصواب أو خطأ.

| الحقل | الوصف |
|------|------|
| `statement` | نصّ العبارة (يُعرض مكان `prompt`) |
| `answer` | `true` إن كانت العبارة صحيحة، `false` إن كانت خاطئة |

```js
{
  type: "true-false",
  objective: "4Bh7: يستنتج أنّ العظام ترتبط لتكوّن الهيكل العظمي",
  statement: "الجُمجمة تتكوّن من عظمة واحدة فقط.",
  answer: false
}
```

## ⑤ تحديد الأجزاء — `hotspot`
الطالب ينقر على الموضع الصحيح في صورة/رسم.

| الحقل | الوصف |
|------|------|
| `image` | مسار الصورة (أو `svg`) — تُوضع دائماً داخل غلاف يحافظ على نسبتها |
| `bg` | (اختياري) لون خلفية الملعب |
| `spot` | `{ x, y, r }` — مركز المنطقة الصحيحة (`x`/`y` نسبة مئوية) ونصف القطر المقبول `r` (نسبة مئوية، الافتراضي 10) |

```js
{
  type: "hotspot",
  objective: "4Bh7: يستنتج أنّ العظام ترتبط لتكوّن الهيكل العظمي",
  prompt: "انقر على القفص الصدري (الأضلاع) الذي يحمي القلب والرئتين.",
  image: "images/skeleton.png",
  fit: "height",
  bg: "#fdf9ee",
  spot: { x: 53, y: 27, r: 11 }
}
```

## نموذج درس كامل (٥ أسئلة، الأنواع الخمسة)
انظر المثال الحيّ في `js/questions.js` تحت المفتاح `"g4s-1-2"` (الهيكل العظمي للإنسان):
صواب/خطأ → اختيار من متعدد → توصيل → سحب وإفلات → تحديد الأجزاء (متدرّجة الصعوبة).

## كيف تُحدّد نِسَب `x`/`y` على صورة؟
`x` = المسافة من **يسار** الصورة ÷ عرضها × 100.
`y` = المسافة من **أعلى** الصورة ÷ ارتفاعها × 100.
(الغلاف يجعل النِسَب محسوبة على الصورة نفسها لا على الملعب، فتبقى دقيقة على كل الشاشات.)
